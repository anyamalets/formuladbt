# Дизайн-аудит formuladbt.ru через ui-ux-pro-max

Проанализированы: текущий код + результаты доменов `ux` (animation/loading/hover), `ux` (mobile/responsive), `style` (альтернативы Minimalism Swiss).

Стиль и палитра уже зафиксированы (Minimalism Swiss + Editorial Grid, пурпур + зелёный, Lora + Roboto). Не меняем — аудит про доводку.

## 🔴 КРИТИЧНО (висит из синтеза 5 ревьюеров)

### 1. Mobile-меню (бургер) на TopNav
Сейчас на мобильном ссылки навигации (`О команде`, `Программа`, `Что такое ДБТ`, `Кому подходит`) скрыты — есть только `Записаться`. Клиент не может быстро перейти к секции.
**Делать:** `Pressable` иконка-бургер → `<details>` или sheet с 5 пунктами. Touch target ≥48×48px.

### 2. Sticky CTA на мобильном футере
60-70% трафика — мобайл. Кнопка «Оставить заявку» только в hero и форме внизу. Между ними — длинный скролл без CTA.
**Делать:** `position: fixed; bottom: 0` на mobile, одна кнопка `Оставить заявку →`. Hide когда сама форма в viewport.

### 3. Яндекс.Метрика + цели
Без метрики все правки слепые — не узнаем, что работает. Ждём от Ани счётчик (завтра).

## 🟠 ВАЖНО — анимации и микро-интеракции (сейчас ноль)

Сайт «снэпает» — состояния меняются мгновенно. Это для серьёзной ниши не критично, но добавляет ощущение «недоделано».

### 4. Анимация раскрытия FAQ-аккордеона
Сейчас `<details>` скачет (height: auto → height: full). Дёргано.
**Делать:** `transition: grid-template-rows 200ms ease-out` или CSS `interpolate-size: allow-keywords` (новый стандарт). При `prefers-reduced-motion` — отключить (уже глобально).

### 5. Hover-эффекты на FAQ-summary
Сейчас при наведении на вопрос — ничего не происходит, не понятно что кликабельно.
**Делать:** `hover:bg-[var(--color-surface-muted)]` + `cursor-pointer`.

### 6. Press feedback на CTA-кнопках
Сейчас при клике на кнопку: цвет меняется на hover-цвет, но без `active:` состояния. Активный нажим не виден.
**Делать:** `active:scale-[0.98]` или `active:bg-[var(--color-primary-darker)]` — лёгкая обратная связь на нажим (правило `scale-feedback` 0.95-1.05).

### 7. Form: фокус-стейт более заметный
Сейчас focus меняет border-color на primary — едва заметно.
**Делать:** добавить `focus:ring-2 ring-[var(--color-primary)] ring-offset-2` — отчётливая «обводка» при tab-навигации.

### 8. Появление команды при скролле (stagger)
Карточки команды одновременно появляются — выглядит как блок данных, не «команда людей».
**Делать:** `IntersectionObserver` + анимация opacity 0→1, translateY 20px → 0 с задержкой 50ms между карточками (правило `stagger-sequence`). Только если нравится — у некоторых это «лишний шум».

## 🟡 СРЕДНЕ — форма и состояния

### 9. Inline-валидация полей формы
Сейчас ошибка одна — общая снизу под кнопкой. Не понятно какое поле невалидно.
**Делать:** `aria-invalid` + красная подкраска границы конкретного поля + ошибка под полем («Введите имя», «Укажите контакт»).

### 10. Toggle Telegram/Phone — segmented control
Сейчас две отдельные кнопки. Не очевидно что это переключатель.
**Делать:** обёрнуть в общий контейнер с одной рамкой, активная заполнена, неактивная прозрачная. Visual cue что «выбор из двух».

### 11. Success-state формы — продолжение пути
Сейчас после отправки: «Заявка отправлена / Координатор напишет в Telegram...». Не видно «что дальше».
**Делать:** добавить ссылку «Вернуться к началу» или кнопку «Прочитать ещё про программу» (якорь #about-dbt).

### 12. Loading-state кнопки Отправить заявку
Сейчас: «Отправляем заявку...» (текст). Можно добавить spinner SVG слева для визуального подкрепления (правило `loading-buttons`).

## 🟢 ПОЛИРОВКА

### 13. Mobile: padding hero py-24 → py-12
Сейчас hero на mobile = 96px сверху и снизу. На iPhone SE (375×667) CTA уходит под сгиб. Уменьшить mobile padding.

### 14. CrisisBox — sticky-кнопка в углу
Лендинг-дизайнер предложил: пользователь в кризисе может пропустить блок. Маленькая кнопка «Кризисная помощь» в правом нижнем углу на mobile (выше sticky CTA), всегда видна.
**Спорно для нашей этики:** может создать ощущение «всё про кризис, мне страшно». Я бы скорее **не делала**.

### 15. Иконки: единый набор
Сейчас в коде есть пара inline SVG (CheckIcon в Criteria, AlertIcon в Crisis) — нарисованы вручную. Toggle Telegram/Phone — кнопки без иконок.
**Делать:** установить Lucide или Heroicons, поменять inline SVG на один импорт, добавить иконки на Telegram/Phone toggle.

### 16. Длина строки в About-DBT/Duration
Сейчас max-w-3xl (~95 символов на строку). Оптимум 60-75.
**Делать:** `max-w-prose` (~65ch) — текст читается заметно легче.

### 17. OpenGraph картинка
В layout.tsx у `metadata.openGraph` нет `images`. При шаре в Telegram/VK будет некрасивое превью (или пустое).
**Делать:** сгенерировать 1200×630 PNG (палитра + название). Или dynamic OG через Next.js `app/opengraph-image.tsx`.

### 18. Schema.org JSON-LD
Сейчас нет. Маркетолог рекомендовал `MedicalBusiness` + `Person` для каждого специалиста + `FAQPage`. Это +10-20% органического CTR без копирайта.

### 19. sitemap.xml + robots.txt
Сейчас нет. Минимум для индексации.

### 20. Скелетоны для портретов команды
Сейчас при медленном интернете портреты появляются через секунду — без placeholder = layout shift.
**Делать:** placeholder через `next/image blurDataURL` (Next.js встроенный механизм).

## Стили, которые рассмотрели и НЕ взяли

| Стиль | Почему не подходит |
|---|---|
| Bento Grids (Apple-style) | Карточные сетки красивы, но для линейного лендинга «о программе» избыточны. Можно частично взять (сетка команды уже Bento-подобная) |
| Neumorphism | Soft 3D эффекты — для health/meditation. У нас серьёзная клиническая ниша, не подходит |
| Minimalist Monochrome | Чисто чёрно-белый, austere. Конфликтует с тёплой пурпурной палитрой |
| Exaggerated Minimalism | Гигантская типографика (clamp 3rem-12rem). Слишком «дизайнерский» вид для клинической ниши |
| Zero Interface | Voice/gesture-first. Не наш формат |
