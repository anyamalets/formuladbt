# Синтез 5 ревью — приоритизированный план правок

Источники: клиент с ПРЛ/РПП/БАР, ДБТ-терапевт-коллега, маркетолог, лендинг-дизайнер, копирайтер.

---

## 🔴 КРИТИЧНО — репутация и большие risk-mitigations

### 1. Репозиционирование «ДБТ-команда» → «команда в подготовке»

**Голос:** ДБТ-коллега (резко)
**Проблема:** сейчас сайт продаёт «полную программу ДБТ» от «ДБТ-команды», но команда ещё не сертифицирована Behavioral Tech. Сертифицированные коллеги в РФ это заметят и не направят клиента (особенно с активной суицидальностью).
**Правка:** в Hero/Team/везде — «команда в подготовке по ДБТ» с явным раскрытием «не сертифицированы DBT-Linehan Board, проходим интенсив Behavioral Tech (часть 2 завершаем в октябре 2026), работаем под супервизией [имя]»
**Эффект:** теряем маркетинговый удар, выигрываем доверие коллег и долгосрочную репутацию
**Решение Ани:** требуется (стратегическое)

### 2. «Индивидуальная ДБТ-терапия» → «ДБТ-информированная индивидуальная терапия»

**Голос:** ДБТ-коллега
**Проблема:** по Лайнен ДБТ требует 4 модусов. Без группы навыков, телефонного коучинга и команды — это **не ДБТ**. Сейчас сайт это маркирует как «один из двух форматов ДБТ» = размывание метода
**Правка:** переименовать второй формат во всех текстах
**Решение Ани:** требуется

### 3. Цикл навыков: 24 vs 48 недель

**Голос:** ДБТ-коллега
**Проблема:** стандарт Лайнен — 48 недель (2 круга), у вас 24. Это либо реальное сокращение, либо упрощение в копи
**Правка:** уточнить и явно написать на сайте «один круг = 24 недели, при необходимости повторяем второй круг»
**Решение Ани:** уточнить факт

## 🟠 ВАЖНО — конверсия, UX, ясность

### 4. H1 — оффер вместо бренда

**Голоса:** маркетолог + лендинг-дизайнер (consensus)
**Сейчас:** H1 = «Формула ДБТ» (бренд никто не ищет)
**Стало:** «Команда ДБТ-психологов для работы с эмоциональной нестабильностью, ПРЛ и РПП». Бренд переезжает в TopNav
**Усилия:** 30 мин

### 5. Hero subtitle сократить + добавить второй «мягкий» CTA

**Голоса:** клиент + копирайтер + лендинг-дизайнер
**Сейчас:** длинный subtitle с тремя проблемами + только один CTA «Записаться» (клиентка с ПРЛ: «слишком рано, я ещё ничего не прочла»)
**Стало:** короткий subtitle + два CTA — primary «Оставить заявку» + ghost «Узнать, подходит ли мне» (якорь на Criteria)
**Усилия:** 30 мин

### 6. Цена и формат полосой сразу под Hero

**Голоса:** клиент + маркетолог
**Проблема:** «не понимаю, по карману ли»
**Стало:** «3000–5000 ₽ за сессию · программа от 12 мес · онлайн» + диапазон месячной суммы за полную программу
**Усилия:** 30 мин

### 7. Перестановка блоков: Team в конец

**Голоса:** клиент (implicit) + лендинг-дизайнер
**Сейчас:** Hero → Team → AboutDbt → ...
**Стало:** Hero → AboutDbt → Criteria → Program → Duration → **Team** → CrisisBox → AfterApplication → Form → FAQ → Contacts → Footer
**Логика:** клиент сначала квалифицирует себя, потом смотрит «к кому конкретно»
**Усилия:** 5 мин (правка page.tsx)

### 8. Карточки команды: 4 → 3 в ряд + выровнять высоту + теги «работает с самоповреждением / суицидальным анамнезом»

**Голоса:** клиент + лендинг-дизайнер
**Сейчас:** 4-в-ряд при 1200px = ~270px ширина, «пересушенные CV-шники»; разная высота из-за разной длины focus; не видно «кто берёт мой набор»
**Стало:** 3-в-ряд, `h-full flex flex-col + mt-auto` для выравнивания цены; добавить явные теги-чипы вверху карточки: «РПП», «ПРЛ», «Самоповреждение», «Суицидальный анамнез», «БАР»
**Усилия:** 2 часа

### 9. Форма: подсказка «можно не писать диагноз, обсудим в переписке»

**Голос:** клиент
**Проблема:** «не могу написать в форму незнакомому человеку про попытку суицида и порезы»
**Усилия:** 10 мин

### 10. Mobile-меню (бургер) + sticky CTA на мобильном

**Голоса:** маркетолог + лендинг-дизайнер
**Сейчас:** на mobile навигация спрятана без бургера; CTA только в Hero и форме (60-70% трафика — мобайл)
**Усилия:** 2 часа

### 11. Метрика + цели в Yandex.Metrika

**Голос:** маркетолог
**Проблема:** аналитики нет вообще — Я.Директ слепой
**Установить:** счётчик (без вебвизора — этика), цели: submit формы, клик в Telegram, скролл 50/75/100, клик в кризисный блок (отдельная цель, исключить из конверсий)
**Усилия:** 1 час

## 🟡 КОПИРАЙТ — точечные правки (минус AI-ритм, минус хеджи)

### 12. Удалить «без паники»

**Голоса:** клиент + ДБТ-коллега + копирайтер (CONSENSUS-3)
**Где:** Criteria-3 «На программе с этим работают, без осуждения и без паники» → «...спокойно и без осуждения»

### 13. Официальная терминология навыков ДБТ

**Голоса:** ДБТ-коллега + копирайтер + правило памяти `feedback_dbt_skill_terminology`
**Где:** Program → «Тренинг навыков»
- «осознанность» → ОК
- «эмоциональная регуляция» → «навыки регуляции эмоций»
- «межличностная эффективность» → «навыки межличностной эффективности»
- «стрессоустойчивость» → «навыки перенесения дистресса»

### 14. Убрать «во время паники» подобные триггеры в Criteria

**Голос:** копирайтер + клиент
- Criteria-1: «кажется неуправляемым» → «ощущение управления теряется»
- Criteria-6: «именно с такими ситуациями» → «изначально создана для работы с такими случаями»
- Criteria-7: «Вы уже были у одного, двух, трёх специалистов» (rule of three) → «За плечами уже несколько терапевтов и подходов»

### 15. FAQ Q5: смягчить «4 пропуска = вылет»

**Голос:** клиент с ПРЛ (триггерно)
**Сейчас:** жёсткий тон «выбывание из программы»
**Стало:** добавить мостик: «При обострениях правила применяются мягче — обсуждается индивидуально с терапевтом»
**Условие:** правда ли это для вашей команды? Нужно подтверждение Ани

### 16. FAQ Q7: смягчить тональность

- «Точных сроков нет, и осторожно отнеситесь к обещаниям типа „через 3 месяца"» → «Точных сроков нет; к обещаниям „через 3 месяца станет легче" стоит относиться с осторожностью»

### 17. FAQ Q8: убрать конструкцию «не X, а Y» (стоп-лист)

- «Цель команды — не „обсудить клиента", а помочь терапевту работать лучше» → «Команда обсуждает не клиента как личность, а задачи терапевта в работе»

### 18. Дубли формулировок про оплату

**Голос:** копирайтер
**Проблема:** «каждый терапевт ведёт практику самостоятельно» × 4 раза (Program, Duration, AfterApplication, FAQ)
**Решение:** оставить каноническую формулировку в Duration, в других местах — короткое «как устроена оплата»

### 19. Form: button + success + dedupe disclaimer

- «Записаться» → «Отправить заявку» (точнее: координатор пишет, не сессия)
- Success: «Координатор Анна напишет в Telegram. По выходным — в понедельник утром.»
- Убрать дублирующий дисклеймер под кнопкой (уже есть в чекбоксе)

### 20. Биосоциальная модель: расширить «обесценивающее окружение» за пределы родителей

**Голос:** ДБТ-коллега
**Сейчас:** примеры только семейные («годами реагировало... не плачь»)
**Стало:** добавить «школа, медсистема, культура, травма» — для клиентов с «хорошими родителями» это спасает от «модель не про меня»
